# Pasty 0.8.5-beta

ノッチが「降りてくる」を超えて、ついに「**そこに在る**」レベルに到達しました。

## ⚡️ ノッチ「瞬間表示」へ

- v0.8.4 で残っていた hover→show の **マイクロ遅延を全撤去** しました
- ノッチ起動の `dwellDelay` を 80ms → **0ms** (即時) に変更
- パネルが降りてくる **アニメーション (120ms) を撤去** し、最終位置に一発配置
- 起動時の prewarm で `makeKey()` と マウス/キー モニタも事前装備 → 初回 hover でも「初回コスト」がゼロ
- `scheduleShow` の `DispatchWorkItem` 経由を 0ms 時に skip し、同じ run-loop tick で `show()` を叩く

## 🎚 ノッチ起動遅延スライダー

Settings → サーフェス に 2 つのピッカーを追加しました。

- **ノッチ起動遅延**: 0ms (即時) / 50ms / 100ms / 200ms。デフォルト **0ms**
- **ノッチアニメ**: 0ms (瞬間表示) / 60ms / 120ms。デフォルト **0ms**

「ふっと触れただけで開く」が気になる方は 50ms / 100ms に上げてください。アニメだけ戻したい場合は 60ms / 120ms を選べます。

## 🐛 残っていた遅延要因の根治

- 初回 show 時に `panel.makeKey()` を毎回払っていたのを prewarm に移動
- `installMouseMonitors` を起動時の `installPersistentMonitors` に変更し、show ごとに `addGlobalMonitorForEvents` ×3 を呼ぶコストを削除
- `show()` の `orderFrontRegardless` 直前に `layoutSubtreeIfNeeded` を再度呼び、初回 real show の SwiftUI 再評価コストを潰す
- `dismiss()` も anim=0 時に `setFrame` を skip して `orderOut` 一発で閉じる

---

体感が変わらない場合は **Settings → サーフェス → ノッチ起動遅延 = 0ms / ノッチアニメ = 0ms** に設定されているか確認してください。
