# Pasty 0.9.3-beta — フォルダ並べ替え回帰ホットフィックス

## 🛠 フォルダ並べ替えの青い縦線を完全復活

- v0.9.2 で再発していた「フォルダタブをドラッグしても青い縦線インジケータが出ず、並べ替えが効かない」回帰を修正
- 原因は `folderTab` の SwiftUI モディファイア順回帰: `.acceptClipReferenceDrop` が `.draggable` の外側に積まれてしまい、ギャップ側 `PinboardDragItem` の `dropDestination` が反応しなくなっていた
- v0.9.1 のチェーン (`.draggable(PinboardDragItem)` を最外側) に戻し、タブ間ギャップでのホバー検知と `PinboardStore.reorder` 発火を v0.8.7 / v0.8.8 / v0.9.1 と同じ感触で復活

## 🛠 ドラッグプレビューを再び不可視化

- ドラッグ開始時にカーソルへ追従するシステム由来のタブスナップショットを廃止
- プレビューを 1×1 の完全透明な `Rectangle` に置き換え、macOS ネイティブのドラッグカーソルだけが表示されるように
- ドロップ先のフォルダタブが常に視認でき、青い縦線インジケータとの干渉もなくなりました

## ✅ 半透明フェードは引き続き有効

- v0.9.2 で導入した「ドラッグ中の元タブを opacity 0.2 にフェード」は維持
- ドロップ完了時もキャンセル時 (Esc / ウィンドウ外でリリース) も、opacity は 1.0 に復帰します
- フェードのトリガとなる `draggedBoardId` はプレビュー内部の `onAppear` で立て、reorder 確定 or キャンセル時には folderDropGap / 後段の onDrop ハンドラ側で nil 化されます

## 影響範囲

- `folderTab` の SwiftUI モディファイアチェーンのみ変更 (順序の入れ替え + ドラッグプレビュー差し替え)
- クリップ単発 / 複数選択ドラッグ・他アプリへのテキストドロップ・`Transferable` ペイロード経路は一切変更なし
- `folderDropGap.dropDestination` / `PinboardStore.reorder` のロジックには変更なし
