# v0.9.4 フォルダ並べ替え D&D 再設計

> **対象**: `Sources/Pasty/StripPanel.swift` の `inlineFolderStrip` 周辺 + `Sources/Pasty/DragDropSupport.swift`
> **ゴール**: フォルダタブのドラッグ並べ替えが「4 リリース連続で壊れる」構造を根本から断ち切り、クリップドロップとの相互干渉をモディファイア順に依存させない。

---

### 1. 構造的な再発根本原因

SwiftUI の高レベル D&D API (`.draggable(Transferable)` + `.dropDestination(for: Transferable.self)`) は、内部でビュー階層の **ヒットテスト順** に依存して payload を経路選択する。`ModernFolderTab` 1 枚のタブに対して

1. クリップ受け側: `.acceptClipReferenceDrop` → 内部で `.dropDestination(for: String.self)` (`ClipReceiveDropTarget`)
2. フォルダ並べ替え側: `.draggable(PinboardDragItem)` + ギャップの `.dropDestination(for: PinboardDragItem.self)`

の **2 種類の D&D を同じビュー幹に積層** している。両者は UTType レベルでは分離されているが、SwiftUI のヒットテストは「最も外側のモディファイアが先に hit する」ため、`.draggable` と `.acceptClipReferenceDrop` の **適用順 (= モディファイアスタックの上下)** が一段ずれただけで、タブ本体の clip-drop destination がギャップ側 `PinboardDragItem` の経路を遮り、青い縦線インジケータと並べ替えが死ぬ。

過去の修正履歴がこれを裏付ける:

- `b082027 v0.8.8-beta: split drop targets — folder reorder via gap only, clip drop via tab body only` — UTType を分けただけでは不十分だった初動。
- `33b612a v0.9.1-beta: invisible drag preview + restore folder reorder indicator` — `.draggable` を最外側に置き直してインジケータ復活。
- `f3a3ea1 v0.9.2-beta: fade dragged folder tab to opacity 0.2` — `.opacity` 挿入時に再びモディファイア順が崩れて回帰。
- `0cfe97c v0.9.3-beta: fix folder reorder regression + invisible drag preview` — v0.9.1 のチェーンへ巻き戻し。それでもユーザー報告でまだ壊れている。

つまり「Transferable 型で分ければ衝突しない」前提が SwiftUI のヒットテスト実装に対して **暗黙的かつ脆い** ことが構造的原因。リファクタの度にサイレントに壊れる。

---

### 2. 3 案比較

| 観点 | 案A: SwiftUI 高レベル (Transferable + dropDestination) | 案B: SwiftUI 中レベル (onDrag + onDrop DropDelegate) **[推奨]** | 案C: AppKit NSDraggingSource/Destination |
|---|---|---|---|
| 信頼性 | 低 (4 回連続で破綻) | 高 (DropDelegate がヒットテストを明示制御) | 最高 (フル制御) |
| 実装コスト | 低 (現状維持) | 中 (DropDelegate 1 クラス + UTType 経路差し替え) | 高 (NSView ラップ + Coordinator 再実装) |
| preview 制御 | △ (`.draggable` の closure のみ、システムが上書き) | ○ (`NSItemProvider` + 透明矩形で制御可) | ◎ (NSImage 自由) |
| 既存コードとの互換性 | ◎ | ○ (`acceptClipReferenceDrop` は不変、folder 側のみ差替) | × (SwiftUI 階層への侵襲大) |

### 3. 推奨設計

**案B を採用**。フォルダ並べ替えのみ中レベル API へ降ろし、クリップ側の `Transferable` 経路は不変のまま据え置く。`DropDelegate` で `dropEntered`/`dropExited`/`dropUpdated`/`validateDrop`/`performDrop` を明示実装し、ヒットテストとビジュアル状態遷移をモディファイア順から切り離す。

```swift
struct PinboardReorderDelegate: DropDelegate {
    let targetIndex: Int
    @Binding var folderDropTargetIndex: Int?
    @Binding var draggedBoardId: Int64?
    let onReorder: (Int64, Int) -> Void

    func validateDrop(info: DropInfo) -> Bool {
        info.hasItemsConforming(to: [UTType.pastyPinboardTab])
    }
    func dropEntered(info: DropInfo) {
        folderDropTargetIndex = targetIndex
    }
    func dropExited(info: DropInfo) {
        if folderDropTargetIndex == targetIndex { folderDropTargetIndex = nil }
    }
    func dropUpdated(info: DropInfo) -> DropProposal? {
        DropProposal(operation: .move)
    }
    func performDrop(info: DropInfo) -> Bool {
        guard let provider = info.itemProviders(for: [UTType.pastyPinboardTab]).first else { return false }
        provider.loadDataRepresentation(forTypeIdentifier: UTType.pastyPinboardTab.identifier) { data, _ in
            guard let data, let item = try? JSONDecoder().decode(PinboardDragItem.self, from: data) else { return }
            Task { @MainActor in
                onReorder(item.boardID, targetIndex)
                draggedBoardId = nil
                folderDropTargetIndex = nil
            }
        }
        return true
    }
}
```

`.onDrag` 側は `NSItemProvider()` を生成し、`registerDataRepresentation(forTypeIdentifier: UTType.pastyPinboardTab.identifier, visibility: .ownProcess)` で JSON エンコードした `PinboardDragItem` を載せる。UTType `app.pasty.pinboard-tab` (既存定義を流用) によりクリップ用 `String` 経路 (`acceptClipReferenceDrop`) には UTType レベルで合致しない。**DropDelegate は SwiftUI のヒットテスト解決を経由しないため、モディファイア順の影響を受けない** ことが本案の核心。

### 4. クリップ preview UX 再設計

クリップドラッグの大きなカードプレビュー (`ClipDragCard` 220×100 + シャドウ) は、フォルダタブ上で「タブが見えなくなる」問題があるため、薄い小さな矩形プレビューに置き換える。フォルダドラッグ側 (v0.9.3 で導入した 1pt 透明矩形) は据え置く。

- **クリップドラッグ preview**: 80×24pt の `RoundedRectangle(cornerRadius: 6, style: .continuous)` + `.regularMaterial` + `opacity 0.55`。中央に `Image(systemName: "doc.on.clipboard")` (10pt) + 件数バッジ `<N> 件` (caption2 weight semibold)。`ClipDragCard` を `MiniClipDragPreview` に差し替える。
- **フォルダタブ hover ハイライト**: クリップがフォルダタブ上に来たら `RoundedRectangle.strokeBorder(Color.accentColor, lineWidth: 2)` + `scaleEffect(1.05)` + `animation(.spring(response: 0.3, dampingFraction: 0.7))`。既存の脈動グロー (`ClipReceiveDropTarget` の `pulse`) は維持しつつ、stroke + scale を追加。
- **フォルダタブ ドラッグ preview**: v0.9.3 の 1pt 不可視矩形を維持。並べ替え側だけはシステム既定スナップショットに任せる。

**実装ヒント**: `acceptClipReferenceDrop(pinboardId:pinboards:store:isTargeted:)` に `isTargeted: Binding<Bool>? = nil` を追加。`ClipReceiveDropTarget` 内の `@State private var isTargeted` を `@Binding` 化、もしくは内部 state を `onChange` で外部に伝搬する。`ModernFolderTab` 側で `@State var clipHover: Bool` を保持し、scale/stroke の overlay/animation に渡す。
